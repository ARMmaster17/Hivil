Welcome to HIVE.

HIVE is the new way to write programs that take advantage of cluster computing without ever having to deal with MPI or networking code. The HIVE Infrastructure allows you to dynamically add and remove nodes, even while running a program.

How does it work?

You start by running a script on a computer with HIVE installed. That computer becomes the master node. It will then gather a list of slave nodes on the network that are awaiting a task. The program then looks through the script file for "jobs". A job is simply a chunk of code that can run independently from the other jobs in the script file. The HIVE master node will then send each job to a slave node, which will run the code and return a result. Should a node go offline for any reason, the master node will send the job to a new slave node and tell it to pick up where the other node left off.

How do I get it?

Precompiled program

Click "Download" above to download a .zip file containing the program. Inside the .zip file you will find two folders. One contains the server portion that will go on the master node, the other folder contains the client code which you install to all the computers you wish to act as slave nodes.

Build from source

If you need to build from source, click "Download Source" above. This will download the project files in either a .zip file or a .tar.gz file. You will need Visual Studio 2013. Please note that the license for HIVIL prohibits distributing both unmodified and modified versions of the codebase without permission from the original program author(s).

System Requirements

Master Node

Minimum

  • 2 GHz Dual Core CPU
  • 2 GB DDR3 or better RAM
  • 1 Ethernet interface
  • Windows 2008 R2/Windows 7
  • Microsoft .NET 4.5.1

Recommended

  • 3 GHz Hex Core or better CPU
  • 16 GB DDR3 or better RAM
  • SSD
  • Fiber Optic interface/Dual Ethernet interface
  • Windows 8.1/2012 R2
  • Latest version of .NET

Slave Node

Minimum

  • Intel Pentium III 800 MHz
  • 512 MB RAM
  • Windows XP
  • Ethernet interface

Recommended

  • No more than half as powerful as your master node
